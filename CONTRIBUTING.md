# Contributing to GBEMU43DS

## Getting started
1. Fork the repo and clone your fork
2. Create a branch: `git checkout -b feature/your-feature`
3. Build and test locally before opening a PR

## Code style
- C++17, no exceptions in hot paths
- Please do not write C code as it can get messy fast
- Match existing naming conventions
- No trailing whitespace

## Pull requests
- Keep PRs focused on one change
- Describe what and why in the PR description
- Reference any related issues

## Reporting bugs
Open an issue with:
- ROM/game that triggers it (if applicable)
- Performance issues on some games and clean code that could fix it
- Steps to reproduce
- Expected vs actual behavior
